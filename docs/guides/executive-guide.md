Стратегічний огляд AI-агентів: що це, де застосовувати, які є варіанти.

## Що таке AI-агент

### Визначення

**AI-агент** — програма на базі LLM (Large Language Model), яка може:
- Розуміти запити природною мовою
- Виконувати дії в системах (не тільки відповідати)
- Зберігати контекст між сесіями
- Працювати автономно за розкладом

### Різниця від чат-бота

| Характеристика | Традиційний чат-бот | AI-агент |
|----------------|---------------------|----------|
| Логіка | Скрипт, decision tree | LLM розуміє контекст |
| Можливості | Відповіді за шаблоном | Виконує довільні дії |
| Гнучкість | Тільки передбачені сценарії | Адаптується до нових задач |
| Інтеграція | API для кожного сценарію | Може використовувати будь-які tools |

**Приклад:**
- Чат-бот: "Ваше замовлення #123 в статусі 'Доставляється'" (пошук в БД за шаблоном)
- AI-агент: Аналізує всі замовлення, знаходить аномалії, створює тікет, пише персоналізований лист клієнту

## Два продукти: NanoClaw vs Odoo AI

### NanoClaw (open source)

**Що це:** WhatsApp-бот на базі Claude, який працює на вашому сервері.

| Характеристика | Опис |
|----------------|------|
| Доступ | WhatsApp (мобільний, 24/7) |
| Модель | Claude (Anthropic) |
| Ізоляція | Кожна група — окремий контейнер |
| Кастомізація | Повний доступ до коду |
| Вартість | Claude API (~$15-50/місяць типове використання) |

**Що може зараз:**
- Відповідати на питання
- Шукати в інтернеті
- Читати/писати файли
- Виконувати bash команди (в ізольованому контейнері)
- Планувати задачі за розкладом
- Надсилати повідомлення

**Що НЕ може без доробки:**
- Інтеграція з ERP/CRM (потрібно імплементувати)
- Робота з email (є skill /add-gmail)
- Робота з календарем

### Odoo 19 AI Agents (Enterprise)

**Що це:** Вбудовані AI агенти в Odoo ERP.

| Характеристика | Опис |
|----------------|------|
| Доступ | Веб-інтерфейс Odoo |
| Модель | ChatGPT або Gemini (на вибір) |
| Ізоляція | Права доступу Odoo |
| Кастомізація | Через Studio (no-code) |
| Вартість | Odoo Enterprise + API провайдера |

**Що може:**
- Пошук по Odoo природною мовою
- Створення лідів через AI
- AI Server Actions (автоматизація промптом)
- RAG по завантажених документах

**Налаштування агента в Odoo 19:**
```
AI Agent
├── System Prompt (роль, поведінка)
├── LLM Provider (ChatGPT / Gemini)
├── Response Style (Analytical / Balanced / Creative)
├── Topics (Natural Language Search, Create Leads, etc.)
└── Sources (документи для RAG)
```

### Порівняння

| Критерій | NanoClaw | Odoo AI |
|----------|----------|---------|
| Мобільний доступ | WhatsApp | Тільки веб |
| Глибина інтеграції | Потребує розробки | Вбудована |
| Кастомізація | Необмежена (код) | Studio + промпти |
| Автономність | Scheduled tasks | Automated Actions |
| Ізоляція даних | Контейнери | Права Odoo |
| Vendor lock-in | Немає | Odoo Enterprise |

## Де застосовувати

### CRM та продажі

| Процес | Традиційно | З AI |
|--------|------------|------|
| Кваліфікація лідів | Менеджер оцінює вручну | AI scoring за історією |
| Follow-up | Нагадування в календарі | Автоматичні дії за тригерами |
| Звіти | Ручне збирання даних | Запит природною мовою |

**Odoo 19 CRM AI features (з документації):**
- Lead assignment з AI scoring
- Win probability calculation
- Створення лідів з візиток (мобільний додаток)

### Helpdesk

| Процес | Традиційно | З AI |
|--------|------------|------|
| Призначення | Черга або вручну | За тегами, навантаженням |
| Перша відповідь | Шаблон | Персоналізована (з контекстом) |
| Ескалація | Правила | AI аналізує складність |

**Odoo 19 Helpdesk assignment methods:**
- Equal tickets — рівна кількість всім
- Equal open tickets — балансування активних
- Tags-based — за експертизою

### Управління проектами

| Функція | Опис |
|---------|------|
| Smart Forecasting | Прогноз workload (Odoo 19) |
| Task Dependencies | Автоматичний sequencing |
| Status Updates | Агент збирає статуси (NanoClaw scheduled task) |

## Контроль та ризики

### Що AI НЕ гарантує

1. **Галюцинації** — LLM можуть генерувати неправдиву інформацію
   - Odoo "Restrict to Sources" зменшує, але не усуває ризик
   - Критичні рішення потребують human review

2. **Консистентність** — різні відповіді на однакові питання
   - Temperature параметр впливає на варіативність
   - Для критичних процесів — детальні промпти

3. **Безпека** — prompt injection, data leakage
   - NanoClaw: контейнерна ізоляція
   - Odoo: права доступу користувача

## Метрики (приклади, не гарантії)

> **ВАЖЛИВО:** Реальні результати залежать від якості імплементації, даних, і специфіки бізнесу.

| Метрика | Що вимірювати | Типовий baseline |
|---------|---------------|------------------|
| Response Time | Час до першої відповіді | Залежить від процесу |
| Automation Rate | % задач без людини | Почніть з 10%, масштабуйте |
| Error Rate | Помилки AI vs людини | Порівняйте до/після |
| Time Saved | Години на рутину | Виміряйте конкретні задачі |

### Як вимірювати ROI

```
1. Baseline (2-4 тижні)
   - Скільки часу йде на задачу X
   - Скільки помилок
   - Скільки коштує (година × зарплата)

2. Pilot (1-3 місяці)
   - Та сама задача з AI
   - Ті самі метрики

3. ROI = (Baseline Cost - Pilot Cost - AI Cost) / AI Cost × 100%
```

## Рекомендації

### Почати з (низький ризик)

1. **Інформаційні запити** — "Який статус проекту X?"
2. **Нагадування** — автоматичні follow-up
3. **Внутрішні звіти** — summary для команди

### Уникати на початку (високий ризик)

- Фінансові рішення без human approval
- Юридичні документи
- Комунікація з клієнтами без review
- Критичні бізнес-процеси

### План впровадження

```
Фаза 1: Proof of Concept (1-2 тижні)
├── Вибір 1 простого use case
├── Технічна інсталяція
└── Тестування командою

Фаза 2: Pilot (1-3 місяці)
├── Розширення на відділ
├── Збір метрик
└── Ітерації на промптах

Фаза 3: Scaling (після підтвердження ROI)
├── Інтеграція з ERP
├── Більше use cases
└── Автоматизація критичних процесів (з human-in-the-loop)
```

## Варіанти архітектури

### Варіант A: Тільки NanoClaw

```
WhatsApp ─── NanoClaw ─── Claude
                │
                └── Файлова система (пам'ять, задачі)
```

**Плюси:** Простота, мобільний доступ, повний контроль
**Мінуси:** Немає ERP інтеграції з коробки

### Варіант B: Тільки Odoo AI

```
Odoo Web UI ─── Odoo AI Agent ─── ChatGPT/Gemini
                     │
                     └── Odoo Database
```

**Плюси:** Глибока інтеграція, no-code налаштування
**Мінуси:** Тільки веб, vendor lock-in

### Варіант C: NanoClaw + Odoo (потребує розробки)

```
WhatsApp ─── NanoClaw ───┬─── Claude
                         │
                         └─── Odoo API ─── Odoo Database
                                              │
                         Odoo Web UI ─── Odoo AI Agent
```

**Плюси:** Мобільний доступ + глибока ERP інтеграція
**Мінуси:** Потребує розробки інтеграції
