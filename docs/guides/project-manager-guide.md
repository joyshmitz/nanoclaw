## Scheduled Tasks (працює зараз)

### Створення задачі

```
@Мікі щодня о 9:00 надсилай мені повідомлення "Час на standup"
```

**Параметри schedule_task:**

| Тип | Формат | Приклад |
|-----|--------|---------|
| cron | `"хв год день місяць день_тижня"` | `"0 9 * * 1-5"` (9:00 пн-пт) |
| interval | мілісекунди | `"3600000"` (кожну годину) |
| once | ISO timestamp (локальний час) | `"2026-02-15T10:00:00"` |

### Context modes

| Режим | Коли використовувати |
|-------|---------------------|
| `group` | Задача потребує контексту розмови |
| `isolated` | Самодостатня задача (включіть весь контекст в промпт) |

**Приклади:**

```
# group — потрібен контекст
@Мікі щопонеділка о 9:00 нагадай мені про те, що ми обговорювали минулого тижня

# isolated — самодостатня
@Мікі щодня о 18:00 перевір weather API і повідом якщо буде дощ завтра
```

### Керування задачами

```
@Мікі покажи всі мої scheduled tasks
@Мікі призупини задачу {id}
@Мікі відміни задачу {id}
```

## Пам'ять (CLAUDE.md)

### Як працює

Кожна група має файл `groups/{folder}/CLAUDE.md`, який Claude читає при кожному запиті.

### Що туди записувати

```markdown
# Project Context

## Поточний спринт
- Sprint 5: 1-15 лютого
- Головна ціль: MVP checkout flow

## Команда
- Олег — backend
- Марина — frontend
- Сергій — QA

## Важливі дати
- 15 лютого — demo для клієнта
- 20 лютого — реліз v1.0

## Мої преференції
- Відповідай коротко
- Використовуй українську
```

### Запис в пам'ять

```
@Мікі запиши в CLAUDE.md що дедлайн перенесли на 20 лютого
```

## Практичні сценарії

### Сценарій 1: Щоденний standup reminder

```
@Мікі створи scheduled task:
- Щодня о 9:00 з понеділка по п'ятницю
- Надсилай повідомлення "Час на standup! Що зробив вчора? Що плануєш сьогодні?"
- Режим isolated
```

### Сценарій 2: Моніторинг репозиторію

```
@Мікі створи scheduled task:
- Кожні 4 години
- Запусти git log --oneline -5 і надішли мені summary нових комітів
- Режим group (щоб мати контекст проекту)
```

### Сценарій 3: Weekly digest

```
@Мікі щоп'ятниці о 17:00:
- Прочитай всі файли в директорії reports/
- Створи summary за тиждень
- Надішли мені
```

### Сценарій 4: Пошук інформації

```
@Мікі знайди в інтернеті:
- Останні best practices для React Server Components
- Сформуй короткий список з посиланнями
```

### Odoo Automation Rules (налаштовується в Odoo, не в NanoClaw)

**Що можна налаштувати в Odoo Studio:**

| Тригер | Умова | Дія |
|--------|-------|-----|
| Task created | Priority = High | Send webhook |
| Task overdue | — | Send email |
| Stage = Done | Project = X | Update related records |

**AI Server Actions (Odoo 19):**

```
Промпт: "Коли задача прострочена більше 2 днів:
1. Змінити priority на High
2. Відправити email assignee"
```

Це налаштовується в Odoo Studio → Automated Actions → AI Server Action.

## Helpdesk (Odoo 19)

### Assignment methods

| Метод | Опис | Коли використовувати |
|-------|------|---------------------|
| Equal tickets | Рівна кількість всім | Однакова складність тікетів |
| Equal open | Рівна кількість відкритих | Різний час обробки |
| Tags-based | За експертизою | Спеціалізовані агенти |

### SLA Rules

```
Priority High: 1 година на відповідь
Priority Medium: 4 години
Priority Low: 24 години

On SLA breach:
├── Escalate to Team Lead
└── Send notification
```

### Stage-based notifications

| Перехід | Дія |
|---------|-----|
| New → In Progress | Notify customer: "Working on it" |
| In Progress → Waiting | Notify customer: "Need info" |
| Waiting → Resolved | Notify customer + survey |

## Tips & Tricks

### 1. Шаблони в CLAUDE.md

```markdown
## Мої запити

### Daily check
Покажи:
- git status
- останні 3 коміти
- чи є uncommitted changes

### Code review
Прочитай {file} і дай feedback:
- Чи є code smells
- Чи є security issues
- Що можна покращити
```

**Використання:**
```
@Мікі запусти мій "daily check" шаблон
```

### 2. Контекст в промпті

```
# Погано — немає контексту
@Мікі виправ баг

# Добре — є контекст
@Мікі в файлі src/utils/validation.ts функція validateEmail
повертає true для невалідних email типу "test@". Виправ регулярку.
```

### 3. Batch операції через bash

```
@Мікі виконай:
for f in src/**/*.ts; do echo "=== $f ==="; head -5 "$f"; done
```

### 4. Перевірка scheduled tasks

```
@Мікі покажи всі задачі і коли вони наступного разу запустяться
```

## Обмеження

### NanoClaw

| Обмеження | Причина | Workaround |
|-----------|---------|------------|
| Немає ERP інтеграції | Не імплементовано | Розробити MCP tools |
| Тільки WhatsApp | By design | Додати канал через skill |
| Немає веб-інтерфейсу | By design | WhatsApp = інтерфейс |
| Контейнер timeout | 10 хв за замовчуванням | Змінити в config |

### Claude (загальні)

| Обмеження | Причина | Workaround |
|-----------|---------|------------|
| Галюцинації | Природа LLM | Перевіряти критичну інформацію |
| Context window | Технічне | Session compaction (автоматичне) |
| Не real-time | API latency | Не для критичних систем |
